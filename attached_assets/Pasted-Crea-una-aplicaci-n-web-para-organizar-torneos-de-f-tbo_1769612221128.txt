Crea una aplicación web para organizar torneos de fútbol (liga) con autenticación, roles y paneles. Debe ser full-stack, responsive, en español, y lista para ejecutar en Replit.

## Stack requerido
- Frontend: React + Vite + TypeScript (UI limpia y moderna).
- Backend: Node.js + Express + TypeScript.
- Base de datos: SQLite (usando Prisma ORM) para que funcione fácil en Replit.
- Auth: JWT + hash de contraseñas (bcrypt).
- Validación: Zod (en backend).
- Estilos: TailwindCSS.

## Roles y permisos (RBAC)
Existen 3 tipos de usuarios:
1) ADMINISTRADORES (3 cuentas iniciales fijas):
   - Acceso total a toda la app.
   - Pueden crear/editar/eliminar: torneos, equipos, jugadores, partidos, árbitros, usuarios.
   - Pueden corregir resultados y eventos si se necesita.
2) CAPITANES:
   - Pueden dar de alta y editar SOLO su equipo.
   - Pueden registrar datos generales del equipo (nombre, escudo opcional, colores, sede).
   - Pueden registrar jugadores de su equipo (alta/edición/baja).
   - NO pueden editar resultados ni calendario.
3) ÁRBITROS:
   - Pueden ver su calendario de partidos asignados.
   - Pueden subir resultados de los partidos asignados.
   - Pueden registrar eventos del partido:
     - Goles (jugador, minuto, equipo).
     - Tarjetas amarillas/rojas (jugador, minuto).
   - NO pueden crear equipos ni jugadores.

## Reglas funcionales clave
- Un torneo tiene temporadas o ediciones (por simplicidad, 1 torneo activo a la vez está bien).
- Un equipo tiene muchos jugadores.
- Un partido tiene: fecha/hora, jornada, equipo local/visitante, cancha, árbitro asignado, estado (PROGRAMADO, JUGADO).
- Al cargar resultado:
  - Guardar marcador final.
  - Guardar lista de eventos (goles/tarjetas).
  - Calcular automáticamente tabla de posiciones:
    - PJ, PG, PE, PP, GF, GC, DG, PTS.
    - PTS: victoria 3, empate 1, derrota 0.
- Mostrar en portada (página principal pública o semi-pública):
  1) Calendario de partidos (próximos y por jornada).
  2) Tabla de posiciones (orden por PTS, luego DG, luego GF).
  3) Resultados de partidos (últimos jugados con detalle).

## Pantallas requeridas
### Público / Home
- Header con selector de torneo/temporada (si aplica).
- Secciones:
  - Calendario (filtro por jornada y por equipo).
  - Tabla de posiciones.
  - Resultados recientes.
- Al hacer click en un partido: ver detalle con eventos (goles y tarjetas).

### Login
- Email/usuario + contraseña.
- Redirección según rol.

### Panel Admin
- Dashboard con accesos a:
  - Usuarios (crear capitanes y árbitros, reset contraseña).
  - Equipos (CRUD).
  - Jugadores (CRUD global, pero idealmente desde equipo).
  - Partidos (CRUD: crear calendario, asignar árbitro, cancha, fecha, jornada).
  - Torneo/Temporada (CRUD básico).
  - Correcciones de resultados (audit log simple).

### Panel Capitán
- “Mi Equipo”
  - Editar datos del equipo.
  - Gestionar jugadores (alta/edición/baja).
  - Ver calendario de su equipo y resultados (solo lectura).

### Panel Árbitro
- “Mis Partidos”
  - Lista de partidos asignados (PROGRAMADOS y JUGADOS).
  - En partido PROGRAMADO: botón “Cargar resultado”
  - Formulario para:
    - marcador final
    - registrar goles (jugador, minuto, equipo)
    - registrar tarjetas (jugador, minuto, tipo)
  - Guardar y marcar partido como JUGADO.

## Modelo de datos (mínimo)
- User: id, name, email, passwordHash, role (ADMIN | CAPITAN | ARBITRO), teamId opcional (para capitanes), createdAt
- Tournament: id, name, seasonName, active boolean
- Team: id, tournamentId, name, colors, homeField, logoUrl opcional, captainUserId opcional
- Player: id, teamId, firstName, lastName, jerseyNumber, position opcional, active boolean
- Match: id, tournamentId, roundNumber (jornada), dateTime, field, homeTeamId, awayTeamId, refereeUserId, status, homeScore, awayScore
- MatchEvent: id, matchId, type (GOAL | YELLOW | RED), minute, teamId, playerId
- Standing (opcional materializada) o calcular al vuelo:
  - si materializas: teamId, PJ, PG, PE, PP, GF, GC, DG, PTS, updatedAt

## API (mínimo)
- Auth:
  - POST /api/auth/login
  - GET /api/auth/me
- Torneo:
  - GET /api/tournaments/active
- Home:
  - GET /api/home/schedule?round=&teamId=
  - GET /api/home/standings
  - GET /api/home/results?limit=
  - GET /api/matches/:id
- Admin CRUD (protegido role ADMIN):
  - /api/admin/users
  - /api/admin/teams
  - /api/admin/players
  - /api/admin/matches
  - /api/admin/tournaments
- Capitán (role CAPITAN):
  - GET/PUT /api/captain/team
  - CRUD /api/captain/players
- Árbitro (role ARBITRO):
  - GET /api/referee/matches
  - POST /api/referee/matches/:id/result  (incluye marcador + eventos)

## Seguridad / permisos
- Middleware de autenticación JWT.
- Middleware authorizeRoles([...]).
- Validar que:
  - CAPITAN solo toca su teamId.
  - ARBITRO solo toca partidos donde refereeUserId = su usuario.
- Sanitizar inputs y manejar errores claros.
