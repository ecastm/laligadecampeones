ACTÚA COMO: desarrollador full-stack senior.

CONTEXTO:
La plataforma tiene un “Módulo de Marketing” donde ya se guardan imágenes (assets) asociadas a partidos/fechas/equipos o, al menos, con metadatos (nombre, fecha de carga, tags, etc.). Quiero construir una funcionalidad adicional dentro de Marketing llamada “Generador para Redes”.

OBJETIVO (FUNCIONALIDAD CLAVE):
Convertir imágenes del módulo de Marketing en piezas listas para redes sociales, según el formato elegido:
- POST (feed) → imagen 1080x1350 (y opcional 1080x1080)
- HISTORIA → 1080x1920
- REEL → MP4 1080x1920 (6–12 segundos), usando imagen(es) + animación simple

IMPORTANTE:
NO publicar directamente en IG/TikTok.
SÍ generar archivos descargables + texto/copy listo para copiar/pegar + preview en pantalla.

ENTRADA PARA GENERACIÓN (NUEVO REQUISITO):
Además de elegir imágenes, el usuario puede escribir:
- Palabras clave (ej: “Jornada 4”, “semifinal”, “MVP”, “clásico”, “goleada”, “Fair Play”)
- Fecha o rango de fechas de partidos (ej: 2026-03-01 o 2026-03-01 a 2026-03-07)
Con eso el sistema debe:
1) Sugerir/filtrar imágenes del módulo de Marketing relacionadas (por metadatos y/o coincidencia con nombre/etiquetas).
2) Generar automáticamente el contenido (copy + hashtags) consistente con esas palabras/fechas.

REGLA CLARA:
Si NO hay datos del partido en base de datos, NO inventar nombres, marcadores o equipos.
En ese caso, el copy debe quedar “genérico” y pedir que el usuario complete los campos faltantes.
Si SÍ existe info del partido (equipos, fecha, marcador, jornada, cancha), usarla.

FLUJO DE UI (MVP):
Ubicación: Marketing → “Contenido para redes”
Paso 1: Selección de fuente
  - Buscar por:
    a) Palabras clave (input)
    b) Fecha o rango de fechas (date picker)
  - Resultado: Galería de imágenes del módulo Marketing (con checkbox multiselección)
Paso 2: Tipo de contenido
  - POST / HISTORIA / REEL
Paso 3: Plantilla
  - “Marcador”
  - “MVP”
  - “Próximo Partido”
(plantillas simples, funcionales; no diseñar 20)
Paso 4: Campos opcionales/editables (autorrelleno si hay datos)
  - Equipos
  - Marcador
  - Jornada
  - Fecha / hora
  - Lugar
  - MVP / nombre jugador
  - CTA (ej: “Síguenos para más”)
  - Hashtags (autogenerados + editables)
Paso 5: Generar
  - Preview del resultado
  - Botón “Descargar”
  - Caja “Copy listo” + botón “Copiar”
  - Lista de hashtags con botón “Copiar hashtags”

GENERACIÓN DE COPY + HASHTAGS (MVP):
1) Crear un generador de texto basado en plantillas (NO IA externa obligatoria en MVP).
2) Entradas del generador:
   - palabras clave ingresadas
   - fecha(s)
   - datos del partido (si existen en BD)
   - nombres de equipos (si existen)
3) Salidas:
   - Copy para el formato elegido:
     - POST: 1–2 párrafos cortos + CTA
     - HISTORIA: 1 línea fuerte + CTA
     - REEL: 1–2 líneas cortas
   - Hashtags:
     - Base fija: #Futbol #Torneo #Liga #Jornada #Goles #Equipo #Partido
     - Derivados por keyword (map):
       “semifinal” → #Semifinal #Eliminatoria
       “final” → #Final #Campeonato
       “MVP” → #MVP #FiguraDelPartido
       “clásico” → #Clasico
       “fair play” → #FairPlay #Respeto
       “jornada X” → #JornadaX (normalizar)
     - Derivados por equipos (si existen): #NombreEquipo1 #NombreEquipo2 (sanitizados sin espacios)
     - Derivados por ciudad/liga (si existe)
4) Todo debe quedar editable antes de descargar/copiar.

RENDERS / OUTPUT:
POST:
- 1080x1350 (prioridad) y opción 1080x1080
- Si se seleccionan varias imágenes → carrusel: N imágenes renderizadas iguales.
HISTORIA:
- 1080x1920, respetar safe-area
REEL:
- 1080x1920 MP4 (H.264), 24–30fps
- 6–12s configurable
- Imagen + animación simple (zoom suave, paneo, fade)
- Texto overlay según plantilla (marcador, vs, MVP, fecha)

PERSISTENCIA:
Guardar cada render generado en Marketing como “Asset generado” con:
- tipo (post/story/reel)
- plantilla
- keywords usadas
- fecha(s)
- ids de imágenes origen
- copy y hashtags guardados

ENDPOINTS SUGERIDOS:
- GET /api/marketing/assets?keywords=...&from=YYYY-MM-DD&to=YYYY-MM-DD
  -> devuelve imágenes del módulo Marketing filtradas
- GET /api/social/templates
- POST /api/social/generate
  body: {
    type: "post"|"story"|"reel",
    template: "score"|"mvp"|"next_match",
    assetIds: [...],
    keywords: "...",
    dateFrom: "...",
    dateTo: "...",
    overrides: { teams, score, matchday, datetime, venue, mvpName, cta, hashtags }
  }
  -> devuelve { previewUrls, downloadUrls, copy, hashtags }
- POST /api/social/save (opcional si no lo haces dentro de generate)
