Crea una aplicación web full-stack para organizar torneos de fútbol con autenticación, creación de perfiles y selección de roles. Debe ser responsive, en español y lista para ejecutar en Replit.

## Stack requerido
- Frontend: React + Vite + TypeScript
- Backend: Node.js + Express + TypeScript
- Base de datos: SQLite con Prisma ORM
- Auth: JWT + bcrypt
- Validación: Zod
- Estilos: TailwindCSS

## Roles (selección obligatoria al crear perfil)
Al registrarse (crear perfil), el usuario debe seleccionar UNO de estos roles:
1) ADMINISTRADOR
2) CAPITÁN
3) ÁRBITRO

Reglas:
- Un usuario solo puede tener un rol.
- La UI y las rutas deben adaptarse según el rol.
- RBAC estricto: no permitir acciones fuera del rol (validar en backend).

## Accesos por rol
### ADMINISTRADOR
Acceso total a la plataforma, incluyendo:
- Gestión completa de torneo/temporada.
- Gestión de usuarios (crear, editar, desactivar, reset password).
- Equipos (CRUD).
- Jugadores (CRUD).
- Árbitros (asignación a partidos).
- Programación de partidos (sección dedicada):
  - Crear partidos (fecha/hora, jornada, cancha, local, visitante, árbitro asignado).
  - Editar partidos programados.
  - Bloquear edición si el partido ya está JUGADO (solo admin puede corregir).
- Corrección de resultados y eventos (audit log simple).

### CAPITÁN
- Alta/edición de datos generales de su equipo:
  - nombre, colores, sede/cancha, logo opcional
- Alta/edición/baja de jugadores de su equipo
- Ver calendario y resultados (solo lectura)
Restricciones:
- No puede programar partidos.
- No puede cargar resultados.
- No puede modificar equipos ajenos ni jugadores ajenos.

### ÁRBITRO
Pantalla especial para registrar datos del partido (solo partidos asignados a ese árbitro):
- Ver “Mis Partidos” (programados y jugados)
- En partido PROGRAMADO:
  - Registrar resultado final (marcador)
  - Registrar eventos:
    - Goles: jugador, minuto, equipo, (opcional: tipo de gol)
    - Tarjetas: amarilla/roja, jugador, minuto
    - Expulsiones: jugador, minuto, motivo (puede ser por roja directa o doble amarilla)
    - Faltas: jugador (opcional), minuto, equipo, descripción (opcional), severidad (leve/media/grave)
- Guardar y marcar partido como JUGADO
Restricciones:
- No puede crear equipos ni jugadores.
- No puede modificar partidos no asignados.
- No puede editar equipos ni usuarios.

## Portada / Página principal (Home)
Con la información del torneo activo, mostrar:
1) Calendario de partidos (próximos, por jornada, filtro por equipo)
2) Tabla de posiciones (PTS, DG, GF como desempate)
3) Resultados de partidos (últimos jugados)
Al hacer click en un partido:
- detalle del partido con eventos: goles, faltas, tarjetas y expulsiones.

## Flujos de autenticación
- Registro (crear perfil):
  - name, email, password
  - role: ADMIN | CAPITAN | ARBITRO (selector obligatorio)
  - Si role = CAPITAN: el admin posteriormente asigna teamId o el capitán crea su equipo desde “Mi Equipo”.
  - Si role = ARBITRO: queda listo para asignación a partidos por parte de admin.
- Login
- Endpoint /me para recuperar rol y permisos
- Protecciones de rutas en frontend y backend

- Mis Partidos (lista)

  - marcador final
  - tabla para eventos (agregar filas):
    - GOAL: minute, teamId, playerId
    - FOUL: minute, teamId, playerId opcional, severity, notes opcional

    - EXPULSION: minute, teamId, playerId, reason
- Guardar resultado

## Reglas de negocio
- Tabla de posiciones se actualiza al cargar resultados:
  - PJ, PG, PE, PP, GF, GC, DG, PTS
  - PTS: victoria 3, empate 1, derrota 0
- Un partido solo se puede marcar como JUGADO si tiene marcador.
- Evitar duplicidades: mismo jugador no puede tener 2 eventos idénticos mismo minuto+tipo (validar).
- Expulsión puede relacionarse con tarjeta roja o evento independiente (guardar ambos si se desea, pero evitar inconsistencias con validaciones).

## Modelo de datos (mínimo)
- User: id, name, email, passwordHash, role, teamId opcional, createdAt
- Tournament: id, name, seasonName, active
- Team: id, tournamentId, name, colors, homeField, logoUrl opcional, captainUserId opcional
- Player: id, teamId, firstName, lastName, jerseyNumber, position opcional, active
- Match: id, tournamentId, roundNumber, dateTime, field, homeTeamId, awayTeamId, refereeUserId, status, homeScore, awayScore
- MatchEvent:
  - id, matchId, type (GOAL | FOUL | CARD | EXPULSION)
  - minute
  - teamId
  - playerId opcional (para FOUL si no se sabe el jugador)
  - metadata JSON (para severity, notes, cardType, reason, etc.)

Opcional:
- Standing materializado o cálculo al vuelo.

## API mínima
- Auth:
  - POST /api/auth/register (incluye role)
  - POST /api/auth/login
  - GET /api/auth/me
- Home:
  - GET /api/home/schedule?round=&teamId=
  - GET /api/home/standings
  - GET /api/home/results?limit=
  - GET /api/matches/:id
- Admin (role ADMIN):
  - /api/admin/users
  - /api/admin/tournaments
  - /api/admin/teams
  - /api/admin/players
  - /api/admin/matches  (programación + asignación árbitro)
- Capitán (role CAPITAN):
  - GET/PUT /api/captain/team
  - CRUD /api/captain/players
- Árbitro (role ARBITRO):
  - GET /api/referee/matches
  - GET /api/referee/matches/:id
  - POST /api/referee/matches/:id/result  (marcador + eventos)
