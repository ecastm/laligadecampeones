  • Tipos TypeScript inferidos de esquemas

1.3 Flujos Principales

Flujo de Autenticación

  1. Usuario envía credenciales → POST /api/auth/login
  2. Server valida con bcrypt → Genera JWT (24h)
  3. Client almacena token en localStorage
  4. Requests incluyen header: Authorization: Bearer <token>
  5. Middleware auth.ts valida y extrae usuario

Flujo del Partido (Árbitro)

  1. PROGRAMADO → Árbitro ve partido asignado
  2. Árbitro inicia partido → POST /matches/:id/start → EN_CURSO
  3. Árbitro registra eventos (goles, tarjetas) durante partido
  4. Árbitro finaliza → POST /matches/:id/finalize → JUGADO
    • Guarda marcador final
    • Genera multas automáticas por tarjetas
    • Actualiza tabla de posiciones

Flujo de Goleadores

  1. Árbitro registra evento GOAL con playerId
  2. API /api/home/scorers agrega eventos GOAL por jugador
  3. Retorna top 20 ordenados por cantidad de goles

2. CASOS DE USO FORMALES

CU-01: Iniciar Sesión

ActorUsuario (Admin/Capitán/Árbitro)
PrecondiciónUsuario registrado en el sistema
FlujoIngresa email + contraseña → Sistema valida → Redirige a panel según rol
PostcondiciónUsuario autenticado con token JWT válido

CU-02: Gestionar Equipos (Admin)

ActorAdministrador
PrecondiciónSesión activa con rol ADMIN
FlujoCRUD completo de equipos (crear, ver, editar, eliminar)
ReglasEquipo asociado a torneo y opcionalmente a división

CU-03: Gestionar Jugadores (Admin/Capitán)

ActorAdministrador o Capitán (solo su equipo)
PrecondiciónSesión activa
FlujoCRUD de jugadores con datos personales
DatosNombre, apellido, posición, número, DNI, fecha nacimiento, fotos

CU-04: Registrar Resultado de Partido (Árbitro)

ActorÁrbitro asignado al partido
PrecondiciónPartido en estado PROGRAMADO o EN_CURSO
Flujo 1. Iniciar partido (cambia a EN_CURSO)
2. Registrar eventos (goles, tarjetas amarillas/rojas)
3. Finalizar partido con marcador
PostcondiciónPartido JUGADO, multas generadas, posiciones actualizadas

CU-05: Generar Calendario Round-Robin (Admin)

ActorAdministrador
PrecondiciónTorneo con equipos asignados
FlujoSelecciona torneo → Configura ida/vuelta → Genera jornadas automáticamente
AlgoritmoCircle method para round-robin balanceado

CU-06: Consultar Tabla de Posiciones (Público)

ActorCualquier visitante
FlujoSelecciona división → Ve torneo activo → Consulta standings
CálculoPTS (3 victoria, 1 empate, 0 derrota), ordenado por PTS > DG > GF

CU-07: Consultar Tabla de Goleadores (Público)

ActorCualquier visitante
FlujoSelecciona división → Ve pestaña Goleadores
DatosTop 20 jugadores con más goles, foto, equipo

CU-08: Finalizar Torneo (Admin)

ActorAdministrador
FlujoSelecciona campeón → Guarda tabla final → Cambia estado a FINALIZADO
PostcondiciónTorneo archivado en historial con posiciones congeladas
